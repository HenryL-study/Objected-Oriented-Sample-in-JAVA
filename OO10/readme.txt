添加乘客请在Passenger_Monitor  里面的run方法中添加。
默认程序已经添加好了300条随机乘客指令。
你可以继续添加，但是队列中最多同时容纳400条指令。
添加完指令后可以继续调用一系列方法查看出租车状态，以及查看可追踪出租车的状态，代码里有样例，
但是要稍等出租车指令添加完成后才能执行。
所以指令一多延迟会很高。我觉得这样并不能很方便的测试
你可以在我的Taxi类里面添加一些控制台输出来看一看是否正常运行~
（我是这么做的）也可以每用一次addPSG,查看一遍你想看的出租车状态，这样几乎无延迟。

addPSG需要传入两个Index坐标
Index构造函数为两个int类型。
不正确的坐标范围会被忽略。
可以通过sleep来控制添加的时间，不过我觉得意义不大。

关于打开关闭道路的方法：
从控制台实时输入
关闭： 1 1 1  前两个数字是坐标，对应传入的地图上的点，第三个数字为要改的数字。
三个数字以一个空格隔开，有且仅有一个空格。

下面，说明我理解的指导书对于开关边的要求，以防产生不必要的误会：
如果对于地图上已存在的边，只能关闭，不连通的边是不能打开的；
所以，3只能改成1或2；1和2只能改成0；0不能修改；能打开的边只能是之前关闭的边。

恢复：首先输入recover，然后选择要恢复的道路序号

例：
1 1 1
Delete successful!
recover
Now changed 1 paths. Input the num that you want to recover.
0: Index (1,1) 2
0
Recover successful!

关于迭代器的说明
新的出租车类新增了两个方法。可以在Passenger_Monitor对应位置使用。
一个可以得到服务的次数，服务次数从0开始计算，即服务2次，记录的服务次数是1
taxi.iterator(i)会返回第i次服务的迭代器。之后可以用.next()访问每一个经过的点。
0<=i<=服务次数 i>服务次数会返回 null， 强行处理null程序会出错。
这个应该由测试者保证。


程序默认设定是70辆普通出租车和30辆可追踪出租车，可以在Main函数里面修改。
设置出租车的位置可以修改出租车构造函数，默认随机。

可追踪出租车允许行走关闭的道路且关闭的道路不计算车流。
